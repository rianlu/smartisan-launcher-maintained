# 旧桌面应用图标页单列结果化设计

## 目标

- 把旧桌面的“应用图标”页从双列点选页改成“全局规则 + 当前结果列表”
- 图标来源统一为：`改进版图标 > 图标包图标 > 默认图标`
- 顶部在同一页面内完成图标包选择与改进版优先级开关
- 列表展示所有 launcher 应用，而不是只展示已有改进版记录的应用
- 当前版本先不做单应用设置

## 页面结构

### 顶部规则区

- 图标包
  - 行标题固定为“图标包”
  - 右侧显示“未使用”或当前图标包名称
  - 点击后弹出选择列表
- 优先使用改进版图标
  - 保留开关，但语义改为全局优先级开关
- 当前规则说明
  - 只展示当前规则，不承担操作
  - 示例：
    - `当前规则：优先使用改进版图标，未适配时使用图标包`
    - `当前规则：使用图标包，未适配时使用默认图标`

### 应用列表区

- 每行只展示：
  - 当前生效图标
  - 应用名
  - 当前来源说明
- 不再提供左右点击切换
- 不再展示勾选框或双列选择态

## 数据规则

### 基础图标

- 未选择图标包时，基础图标为默认图标
- 选择图标包且命中时，基础图标为图标包图标
- 选择图标包但未命中时，基础图标仍为默认图标

### 最终图标

- 开启改进版优先级且当前应用存在改进版图标时，最终使用改进版图标
- 其他情况使用基础图标

## 技术方案

### 图标包管理

- 新增 `IconPackManager`
- 使用 `LauncherPreferences` 保存当前图标包包名
- 通过扫描已安装应用中是否存在 `appfilter.xml` 识别图标包
- 解析 `appfilter.xml` 中的 `component` 与 `drawable` 映射
- 当前版本按包名匹配图标包图标，不做手动单应用另选图标

### 桌面实际生效

- 在 `Utils.loadIcon` 中接入图标包兜底逻辑
- 当未命中改进版图标时，由默认图标加载链路优先尝试图标包
- 切换图标包或改进版开关后，触发应用图标重载

### 设置页实际生效

- `AppIconsSettingsActivity` 改为展示所有 launcher apps
- 左右双列缓存仍保留两份：
  - 第一份缓存当前基础/最终生效图标
  - 第二份缓存改进版候选图标，仅用于判断当前来源
- 顶部选择图标包后，同步刷新列表和桌面图标

## 范围边界

- V1 支持标准 `appfilter.xml` 图标包
- V1 不支持 `iconback` / `iconmask` / `iconupon`
- V1 不支持单应用自定义
- V1 不支持从图标包中手动挑选具体图标
